Hi! I’m gonna share my Git experience at work here
Target audience: You have a very basic Git experience
Structure: The issues that I had from the very first day at work and the appropriate solutions

Note: The solutions here, might not be the best way of course.

How do I create a branch:

Whenever I want to start working on a new task of course I need to create a new fresh branch from the master usually. The point is, if I’m gonna work on some task on my own then I would create a branch from my personal fork. But if I know that some other colleagues might work on the same issue as well, then it does makes more sense to create a branch from the master Repo.
Normally I create a branch from the webUI but also the CL commands is possible.
then after creating the branch I would go as follows in the CL:

git pull
git checkout ''the name of the branch I created"

What I do after finishing specific task:

Whenever I finish part of my task I do commit with an appropriate message, short and meaningful.
and I go:

git commit “appropriate text to this commit”

after finishing all the task and committing all the related commits(funny sentence tho.!) now I have to push them.
and I go easy:

git push

after pushing all the commits, now from the WebUI (of course could be done from CL) I create a pull request to a person who is in charge of reviewing my code and only after his approval, I can merge it to the master branch.
The last part which I have to send a pull request is usually happens when you are working in a company and an important project is ongoing and not everybody are permitted to merge something to the master. And another idea behind it which is also decided in our company is that you are not permitted to approve your own pull request. so the process would be as follows:

I send a pull request ----> my colleague in charge approves —> then I’m allowed to merge

So the concept and main idea is that, it needs to be always at least two people to merge something to the master branch.

What if I accidentally worked on a wrong branch:

let’s make the scenario this way, assuming that I forgot to change or delete my previous branch and for my new task again I start working on the same one. So in this case in order to save my work, first I need to save them in a stack and it goes:

git stash

then I create a new branch same as I did previously.
so I go:

git pull
git checkout “the name of the new branch created”

Now I need to pop everything that I put into the stack.

git stash pop

Finally I can check my changes with:

git diff

so now I have my changes in my new branch without damaging the other branch.
Note: The ideal way is that after merging a branch to the master then delete it and start working on a new fresh branch
